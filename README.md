# 舆情事理图谱工具
## V1:简化版
### 核心功能模块
* **LLM 交互与文本处理**：
* `call_llm`: 封装了对 DeepSeek API 的调用，支持自定义 System/User Prompt 和参数（如 Temperature）。
* `chunk_text`: 将长文本按固定长度（默认 1200 字符）进行分块，以规避大模型单次输入的上下文长度限制。
* **信息提取与清洗**：
* `extract_from_chunk`: 结合预设的 Prompt，从单个文本块中识别出包含 `cause`（原因）、`relationship_type`（关系类型）和 `effect`（结果）的 JSON 数组。
* `extract_json`: 利用正则表达式从 LLM 返回的文本中过滤掉冗余描述，精准提取 JSON 格式数据。
* **知识标准化**：
* `standardize_entities`: 针对初步提取的三元组进行“二次加工”。它再次调用 LLM 来合并同义词、统一实体表达（例如将“AI”和“人工智能”统一），提高知识图谱的质量。

## V2：简单可视化版本
### v1基础上增加可视化
### 核心功能模块

* **数据加载 (`load_triples`)**：
从 JSON 文件中读取结构化的三元组数据（通常包含 `cause`, `effect`, `relationship_type` 等字段）。
* **图结构构建 (`build_graph`)**：
使用 `networkx` 库创建一个**有向图 (DiGraph)**。将“原因”和“结果”作为图的节点（Nodes），将“关系类型”作为连接它们的有向边（Edges）及其属性。
* **布局算法**：采用 `spring_layout`（弹簧布局），使节点分布更均匀，减少交叉。
* **样式定制**：绘制带箭头的边以体现因果指向性，并标注节点名称和边上的关系标签。
* **中文支持**：通过设置 `SimHei` 字体，确保图表中的中文实体能够正常显示。

## V3：风险路径识别
### 一、 路径识别

程序通过 `networkx` 库构建有向图（DiGraph），并运行了两种互补的路径分析逻辑：

#### 1. 主风险后果链（核心影响力追踪）
* **锚点定位**：首先计算节点的“活跃度”（入度+出度）。得分最高的节点被定义为**风险震中**（如：核心政策变动或重大突发事故）。
* **贪心传导（Greedy Propagation）**：从震中节点出发，算法并不遍历所有分支，而是像“追踪水流主干”一样，在每个分叉口选择**出度（Out-degree）最高**的下游节点。
* *逻辑依据*：出度越高，代表该后果引发的连锁反应越多。通过这种方式，算法锁定了一条**最具扩散潜力的单一传导主线**。

#### 2. 最长风险路径（深度演化分析）
* **拓扑排序与去环**：真实数据中可能存在逻辑循环，算法通过“传递约简（Transitive Reduction）”将图优化为有向无环图（DAG）。
* **深度优先搜索深度**：利用动态规划计算图中路径长度最大的链路。
* *逻辑依据*：这条路径展示了风险从诱因到终局经过的**最长逻辑链条**，揭示了事件演化最深远的因果层次。

### 二、 可视化设计

程序使用 `pyvis` 构建交互式 HTML 报告：

#### 1. 颜色编码策略（Color Coding）

* **风险高亮**：被识别为路径上的节点和边统一标记为**鲜红色（#ff4d4d）**。
* **背景弱化**：非关键路径的节点使用冷蓝色，边使用浅灰色。这种强对比让用户在复杂的因果网络中一眼就能定位到关键线索。

#### 2. 交互式布局算法

* **物理模拟（Barnes-Hut）**：采用引力/弹簧模型，让节点自动排布。互相关联的事件会自动聚集，无关联的事件会彼此排开，直观地呈现出“风险簇”的分布。
* **悬停与探索**：每个节点都带有 `title` 标签，用户鼠标悬停即可查看详细事件描述，解决了长文本在图形界面显示拥挤的问题。





## V4：事件标准化
程序实现了一个**基于语义聚类的事件标准化工具**。它的核心目标是解决知识图谱中“同义异名”的问题（例如将“AI技术进步”和“人工智能的发展”统一），通过“向量化+聚类+大模型总结”的流水线，自动将零散的事件描述归纳为标准的抽象名称。

### 核心功能模块说明

#### 1. 语义向量化 (Embedding)

* **技术栈**：使用本地 `SentenceTransformer` (BGE-large-zh 模型)。
* **功能**：将文本形式的事件描述（cause 和 effect）转化为高维数值向量。这样，语义相近的短语在数学空间上的距离就会更近。

#### 2. 密度聚类 (HDBSCAN Clustering)

* **算法**：采用 **HDBSCAN** 算法。
* **功能**：自动识别向量空间中的稠密区域，将语义相似的事件划分为同一个簇（Cluster）。
* **优势**：不需要预先指定聚类数量，且能自动识别“噪声点”（即无法归类的孤立事件，标记为 -1），保证了标准化的质量。

#### 3. 智能抽象命名 (LLM Renaming)

* **技术栈**：DeepSeek API (`deepseek-chat`)。
* **功能**：对于每一个聚类，程序会将该簇内所有的原始事件描述发送给大模型，请它提炼出一个**统一且简洁的抽象名称**。
* **示例**：将 `["油价上涨", "原油成本增加", "能源价格飙升"]` 统一命名为 `“能源成本上升”`。

#### 4. 数据标准化与导出 (Normalization & Export)

* **三元组对齐**：根据聚类结果，将原始 JSON 中的 `cause` 和 `effect` 替换为大模型给出的标准名称。
* **多格式输出**：
* `normalized_triples.json`：存储标准化后的知识图谱数据。
* `cluster_results.csv`：存储明细表，记录每个原始事件所属的簇 ID 及其标准名称，方便人工校验。
